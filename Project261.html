Instructions

This project is a form generator engine called "Form Generator".

The basis of this project is to provide the user with a simple way to create forms that can simultaneously use different styles. To generate a style, many times different elements are required. For example, there is the element called BUTTON, which generates a button on the form however to be able to have a different design you could use CSS to make the changes. Many times, these changes in CSS fall short when performing the style that is required then other resources are used to generate the desired button, and instead of using BUTTON you could use nested DIVs as well as link elements A.

Based on this, many times when a programmer or form designer needs to do a job, it is very complicated to generate the form because as stated before the design of the form is not only to use CSS but also many times to use more resources or more elements to be able to show the design that is needed, now if a programmer shows a job and the final result does not please the client, then he will need to change the design and sometimes he can spend a lot of time changing the design until it is pleasant to the client.

It is here where the Form Generator comes into play, the basis of the project is the creation of an object (JSON) where a format will be used to create the form. This object will contain the specifications of the form, the different elements that will be needed, the types of elements (text boxes, date or numeric) as well as more specific elements such as the minimum date and the maximum date for data entry. Based on this object the Form Builder uses that base and then draws the desired form.

For the demonstration of this functionality, you must click on the different buttons at the top and you can see how dynamically the form on the right changes style, in some styles it uses labels in others it omits them, to be striking it uses SPAN and in other styles he omits them. From these details the programmer should not worry because the Form Generator would do the hard work.

The best thing about this project is that it could be extensible to any number of designs and it only requires adding the design to the engine and then it would be ready to be used by the programmer, it could even be used in a final project in such a way that The end user could change the design independently.
